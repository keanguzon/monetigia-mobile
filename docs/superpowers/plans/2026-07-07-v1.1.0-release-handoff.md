# v1.1.0 Release Handoff & Architecture Summary

> **For agentic workers:** REQUIRED SUB-SKILL: Read this document to understand the current architectural state of v1.1.0 before attempting any new UI modifications.

**Goal:** Provide a stable, production-ready handoff summary of the v1.1.0 release, detailing the exact UI and versioning decisions finalized to resolve critical bugs and pipeline issues.

**Architecture:** We abandoned experimental `react-native-reanimated` features in favor of native React Native `Animated` APIs to ensure compatibility with `DraggableFlatList`. We also finalized a custom string-based currency mask instead of relying on third-party ATM-style libraries.

**Tech Stack:** React Native, Expo, EAS, GitHub Actions, Native `Animated`, Supabase

---

## 1. Architectural Decisions & Fixes (What Was Done)

### A. Reanimated Rollback (Fixing the Missing Wallets List)
The initial attempt to integrate `react-native-reanimated` caused rendering engine conflicts with `DraggableFlatList`, resulting in completely blank list items on the Accounts/Wallets screen.
**Files Reverted to Native `Animated`:**
- `src/app/_layout.tsx`
- `src/components/ui/AnimatedListItem.tsx`
- `src/components/ui/AuroraBackground.tsx`
- `src/components/ui/FAB.tsx`

*Developer Note:* Do NOT introduce `react-native-reanimated` inside list items that are managed by gesture handlers or draggable lists unless strictly verified on a separate branch.

### B. Standard Slide-up Native Modal (`AddTransactionModal`)
We removed `@gorhom/bottom-sheet` portal wrappers for the Add Transaction screen because they caused unhandled promise rejections and race conditions during database insertions.
- **Current State:** Uses the standard React Native `<Modal>` with native `slide` animation.
- **Loading Check:** Modal cannot be dismissed manually while `isLoading` is true (blocking race conditions during Supabase writes).

### C. Custom Currency Masking (Option A)
The user explicitly rejected "ATM-style" cents input (where typing `100` yields `1.00`).
- **Current Implementation:** A custom `<TextInput>` wrapper inside `AddTransactionModal.tsx`.
- **Behavior:** 
  - Typing `100` shows `100`.
  - Automatically formats with thousands separators (`1,234`).
  - Upon leaving the field (`onBlur`), it automatically appends `.00` or retains exact typed decimals.
- *Developer Note:* Do NOT switch back to `react-native-currency-input` unless specifically requested to restore ATM styling.

## 2. CI/CD & Versioning Pipeline

### Versioning Conflict & Resolution
The App Store (iOS) strictly requires a 3-part versioning string (`[major].[minor].[patch]`) for `CFBundleShortVersionString`. Using a 2-part string like `1.1` results in instant rejection during App Store Connect upload.
- **Final Decision:** The application is locked to **`1.1.0`**.
- **Files Modified:** `app.json`, `package.json`

### Deployment Status
- **Git State:** A clean tag **`v1.1.0`** is currently pushed to the remote `master` branch. The messy intermediate `v1.1` tags have been successfully deleted.
- **EAS Updates (OTA):** 
  - iOS and Android bundles for runtime version `1.1.0` have been built and published on Expo Application Services (EAS).
- **GitHub Actions (APK Release):**
  - Pushing the `v1.1.0` tag automatically triggered `.github/workflows/release-apk.yml`.
  - This workflow builds the standalone Android APK on EAS and attaches it to a new GitHub Release page.
